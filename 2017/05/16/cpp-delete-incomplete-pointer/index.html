发表于 2017-05-16 更新于 2022-07-26

简单版

以下代码编译时会有warning：

在GCC4.1.2下，编译出错信息是：

这是因为在foo里，编译器看不到X的完整类型，没办法确定两件事情：

  1. X有没有自定义的析构函数（准确的说，有没有non-trivial的析构函数）。
  2. X有没有自定义的operator delete函数。

在不确定这两件事情的情况下，编译器只能按最普通的方式去处理delete x：

  1. 不调用任何析构函数。
  2. 调用全局的operator delete，通常来说就是直接释放内存。
